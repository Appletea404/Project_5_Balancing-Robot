HIGH 주석 dead code uart_bluetooth 최상위 모듈 전체 주석 처리 (53줄) uart_bluetooth.v : 4–56 ▼

파일 상단에 원래 블루투스 echo 모듈(uart_bluetooth) 전체가 // 주석으로 막혀 있습니다. 현재 이 모듈은 어디서도 인스턴스화되지 않으며, 실제로 사용 중인 모듈은 그 아래의 baud_rate / uart_tx / uart_rx 세 개입니다. 주석 블록이 53줄을 차지해 파일 가독성을 크게 저해합니다.

현재 코드 (uart_bluetooth.v, 4–56줄)
// module uart_bluetooth(
//     input  clk,
//     input  reset,
//     input  rx_pin,
//     output tx_pin
// );
//     wire        b_tick;
//     wire [7:0]  rx_data;
//     wire        rx_done;
// ...  (53줄 전체 삭제 대상)
// endmodule
// ────── 이 아래부터가 실제 사용 중인 코드 ──────
module baud_rate ( ... );
module uart_tx   ( ... );
module uart_rx   ( ... );
조치
  • 라인 4~56 전체(주석 처리된 uart_bluetooth 모듈) 삭제
  • 파일 맨 위에 파일 용도 한 줄 주석 추가 권장:
    // UART primitives: baud_rate, uart_tx, uart_rx
✅ 리스크 없음. 주석 코드는 합성·시뮬레이션에 영향을 주지 않습니다. 삭제해도 동작 변화 전혀 없습니다.
HIGH 미사용 신호 방향 지시자 6개 — 계산 후 버려짐 top.v : 313–318, 436–441 ▼

dir_front, dir_back, dir_left, dir_right, dir_up, dir_down 6개 신호는 ax_corr / ay_corr / az_corr에서 계산되지만 어떤 출력 포트에도, 어떤 서브모듈에도 연결되어 있지 않습니다. Vivado는 이들을 모두 최적화로 제거하지만, 코드에는 남아 혼란을 줍니다.

BEFORE — 삭제 대상
// top.v: 313~318 선언
wire dir_front;
wire dir_back;
wire dir_left;
wire dir_right;
wire dir_up;
wire dir_down;

// top.v: 436~441 assign
assign dir_front = (ax_corr >  TILT_TH);
assign dir_back  = (ax_corr < -TILT_TH);
assign dir_right = (ay_corr >  TILT_TH);
assign dir_left  = (ay_corr < -TILT_TH);
assign dir_up    = (az_corr >  Z_TH);
assign dir_down  = (az_corr < -Z_TH);
AFTER — 두 가지 선택
// 선택 A: 완전 삭제 (권장)
// → TILT_TH, Z_TH localparam도
//   함께 삭제 가능


// 선택 B: LED 출력으로 활용
output wire [5:0] dir_leds, // 포트 추가
assign dir_leds = {dir_front, dir_back,
                   dir_left,  dir_right,
                   dir_up,    dir_down};

함께 삭제 가능한 localparam: TILT_TH = 16'sd1000 (line 310), Z_TH = 16'sd12000 (line 311) — dir_* 외에 사용되지 않으면 삭제 가능.

✅ 리스크 없음. 이 신호들은 어디에도 연결되지 않아 삭제해도 기능 변화 없습니다.
HIGH 미사용 신호 last_err wire — mpu6050_ctrl 출력이 top에서 미사용 top.v : 112 ▼

last_err [2:0]는 mpu6050_ctrl의 오류 코드 출력입니다. top.v에 wire로 선언되어 인스턴스 포트에 연결되어 있지만, 이후 어떤 로직에도, 어떤 출력에도 사용되지 않습니다. 디버그 UART(mpu6050_debug_uart)에도 전달되지 않아 I2C 오류를 확인할 방법이 없습니다.

BEFORE
// 선언: top.v:112
wire [2:0] last_err;

// mpu6050_ctrl 연결 (top.v:496)
.last_err   (last_err),

// 이후 last_err 사용 없음
AFTER — 두 가지 선택
// 선택 A: 포트를 open으로 처리
.last_err   (),  // 미사용


// 선택 B: debug UART로 전달
wire [2:0] last_err; // 유지
// mpu6050_debug_uart에 포트 추가해서
// I2C 오류를 UART 로그에 출력
⚠️ 기능 리스크 없음. 다만 선택 B(debug UART 연결) 권장 — I2C 오류 발생 시 원인 파악이 불가능한 현재 상태를 개선할 수 있습니다.
MEDIUM 런타임 dead 경로 pid.v 속도 댐핑 경로 — KV_DAMP=0으로 항상 dead pid.v : 89–94, 125–126, 163–166, 337–341 ▼

KV_DAMP = 16'sd0으로 설정되어 있어 속도 댐핑 관련 로직 전체가 런타임에 항상 0을 출력합니다. 합성기가 최적화로 제거하지만, v_term_reg, vel_same_dir_reg, v_term_used_w 등의 코드가 남아 읽기 어렵게 만듭니다. 댐핑이 의도적으로 꺼진 상태라면 이를 명시적으로 처리하는 것이 좋습니다.

영향받는 코드 경로
// pid.v:89 — 항상 0인 상수
localparam signed [15:0] KV_DAMP = 16'sd0;

// pid.v:125~126 — KV_DAMP=0이므로 항상 0
reg signed [31:0] v_term_reg;   // 항상 0

// pid.v:163~166 — 항상 32'sd0 반환
wire signed [31:0] v_term_used_w =
    !vel_same_dir_reg ? 32'sd0 :              // 이 분기만 실행됨
    (error_abs_reg <= VEL_DAMP_FULL_ERR_THR) ? v_term_reg : (v_term_reg >>> 1);

// pid.v:337 ST_TERMS — 항상 0 생성
v_term_reg <= $signed(KV_DAMP) * $signed(vel_abs_reg);  // 0 × vel = 0

// pid.v:311 ST_IDLE — vel_same_dir_reg 추적 (항상 무의미)
vel_same_dir_reg <= ...

// pid.v:381 ST_SUM — 항상 -0 == no-op
pid_sum_reg <= p_term_reg + integral - d_term_shaped_w
               - v_term_used_w;   // ← 항상 0, 제거 가능
권장 처리 방법
  • 댐핑을 계속 끌 것이라면: v_term_reg, vel_same_dir_reg, v_term_used_w, ST_TERMS의 v_term 계산, ST_SUM에서 빼는 부분 제거
  • 나중에 다시 켤 가능성이 있다면: KV_DAMP를 파라미터로 승격시키고 generate 블록으로 감싸거나, 주석에 "비활성화됨 — KV_DAMP 조정으로 활성화"를 명시
⚠️ 주의: 댐핑 경로를 삭제하면 나중에 다시 추가하려면 코드를 다시 작성해야 합니다. 현재 PD 튜닝이 진행 중이므로 삭제보다는 `generate` 블록 또는 주석으로 명시하는 방법을 권장합니다.
MEDIUM 런타임 dead 경로 pid.v 부호 반전 플래그 — 항상 기본값 (passthrough wire) pid.v : 92–93, 149–157 ▼

VEL_SIGN_INV = 1'b0, GYRO_SIGN_INV = 1'b0으로 고정되어 있어 조건부 부호 반전 wire가 단순 passthrough가 됩니다. 삼항 연산자가 상수 조건으로 인해 항상 한 방향만 선택되어 dead branch가 발생합니다.

BEFORE (dead branch 포함)
localparam VEL_SIGN_INV  = 1'b0; // 항상 false
localparam GYRO_SIGN_INV = 1'b0; // 항상 false

// 삼항 연산자지만 false 분기 실행 안됨
wire signed [15:0] gyro_aligned_w =
    GYRO_SIGN_INV ? -gyro_in : gyro_in;
wire signed [15:0] vel_aligned_w  =
    VEL_SIGN_INV  ? -vel_in  : vel_in;
AFTER (단순화)
// 플래그/중간 wire 모두 제거
// gyro_aligned_w 대신 gyro_in 직접 사용
// vel_aligned_w  대신 vel_in  직접 사용


// 부호 변경이 필요해지면 top.v에서
// 인스턴스 연결 시 부호 처리하는 것이
// 더 명시적이고 관리하기 편함
⚠️ 실기기 테스트 중 부호 방향 확인이 필요할 수 있으므로, 테스트가 안정화될 때까지 두고 이후 정리하는 것도 합리적입니다.
LOW 주석 dead code angle_calc.v — 수정 전 Stage 1 주석 블록 angle_calc.v : 70–82 ▼

"수정 전" 레이블이 붙은 구 Stage 1 계산 코드(9줄)가 주석으로 남아 있습니다. 바로 아래 "수정 후" 코드로 완전히 대체되었습니다. ACCEL_GAIN, GYRO_GAIN localparam이 추가되면서 구 코드는 더 이상 의미가 없습니다. 

// 수정 전
// // [Stage 1] 데이터 수신 및 1차 곱셈 (무거운 연산 먼저 수행)
// if (data_valid) begin
//     // 수정 전
//     // accel_q8_reg <= $signed(accel_x) * 32'sd90;
//     // 수정 후
//     accel_q8_reg <= $signed(accel_x) * 32'sd60;
//     gyro_q8_reg  <= ($signed(gyro_x) * 32'sd500) >>> 8;
//     calc_step2   <= 1'b1;
// end

// ↓ 이 아래가 실제 사용 코드 (수정 후)
if (data_valid) begin
    accel_q8_reg <= $signed(accel_x) * ACCEL_GAIN;
    gyro_q8_reg  <= ($signed(gyro_x) * GYRO_GAIN) >>> 8;
    calc_step2   <= 1'b1;
end
✅ 리스크 없음. 단순 주석 삭제입니다.
LOW 미사용 신호 top.v — rx_data wire (i2c_master 출력, top에서 미사용) top.v : 57 ▼

rx_data [7:0]는 i2c_master가 마지막으로 읽은 단일 바이트를 출력하는 포트입니다. top.v에서 연결은 되어 있으나 이후 어디서도 사용되지 않습니다. 실제 파싱은 rx_buf [127:0]를 통해 mpu6050_ctrl에서 이루어집니다. 

// top.v 선언
wire [7:0] rx_data;

// i2c_master 연결 — 필요한 포트 연결
.rx_data  (rx_data),   // ← 연결은 됨

// 이후 rx_data 사용 없음 — rx_buf만 사용됨
조치
  • wire [7:0] rx_data; 선언을 유지하거나 포트에 .rx_data()로 open 처리 (Vivado는 미연결 출력 포트를 경고 없이 처리)
  • 삭제 시 i2c_master 인스턴스 연결에서 해당 포트를 ()로 바꿔야 함
✅ 리스크 없음. 단, 나중에 단일 바이트 read가 필요해질 경우 wire를 유지해두는 것도 합리적입니다.
